# Week 14 Lab: Encapsulated Inventory Item

## Overview
Create an Item class that protects Quantity so it can’t go negative.

## Requirements checklist (acceptance criteria)
- [ ] Defines an `Item` class with Name and Quantity
- [ ] Quantity cannot become negative (enforced inside the class)
- [ ] Includes methods like `AddStock(int amount)` and `RemoveStock(int amount)`
- [ ] RemoveStock returns bool to indicate success/failure
- [ ] Main program demonstrates success and failure cases
- [ ] Runs without errors
