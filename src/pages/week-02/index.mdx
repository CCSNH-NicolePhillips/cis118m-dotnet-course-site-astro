## ğŸ—ï¸ The Structural Foundation

Every C# program follows a strict hierarchy:

```
Namespace (Organization)
  â””â”€â”€ Class (Blueprint)
        â””â”€â”€ Method (Action)
              â””â”€â”€ Statements (Instructions)
```

Think of it like a corporation:
- **Namespace** = The company (prevents name collisions with other companies)
- **Class** = A department (contains related functionality)
- **Method** = An employee's job role (performs specific tasks)
- **Statements** = The actual work being done
