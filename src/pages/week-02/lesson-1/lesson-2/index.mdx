## What Are Variables?

A **variable** is a named container that stores data.

Think of it like a labeled box:
- The **label** is the variable name
- The **contents** is the value stored inside

### Why Use Variables?

Instead of writing the same value over and over, you:
1. Store it once in a variable
2. Use the variable name whenever you need that value
3. If the value changes, you only update it in one place

---

## Declaring and Assigning Variables

### Syntax

```csharp
string name = "Nikki";
```

Breaking it down:
- `string` — The **type** (what kind of data: text)
- `name` — The **variable name** (you choose this)
- `=` — The **assignment operator** (stores the value)
- `"Nikki"` — The **value** (the data being stored)
- `;` — Statement terminator

### Common Data Types (for now)

| Type     | What It Stores           | Example                    |
|----------|--------------------------|----------------------------|
| `string` | Text                     | `"Hello"`, `"Nikki"`       |
| `int`    | Whole numbers            | `42`, `-7`, `0`            |
| `double` | Decimal numbers          | `3.14`, `-0.5`, `2.0`      |
| `bool`   | True or False            | `true`, `false`            |

We'll focus on `string` this week. More types next week!

## Using Variables in Output

Once you've stored a value in a variable, you can use it anywhere:

```csharp
string name = "Nikki";
Console.WriteLine(name);  // Prints: Nikki
```

Notice: **No quotes** around `name` when using it. Quotes mean literal text, no quotes means use the variable.

## Special Characters: Newlines

Sometimes you need to print multiple lines from one `Console.WriteLine` statement.

Use `\n` for a **newline** (line break):

```csharp
Console.WriteLine("First line\nSecond line");
```

**Output:**
```
First line
Second line
```

## Combining Text: Concatenation

**Concatenation** means joining strings together using the `+` operator.

```csharp
string firstName = "Nikki";
string lastName = "Smith";
Console.WriteLine("Hello, " + firstName + " " + lastName);
// Output: Hello, Nikki Smith
```

**Rules:**
- Use `+` to join strings
- Include spaces manually: `" "` between words
- Can mix variables and literal text

**Example:**

```csharp
string city = "Portland";
Console.WriteLine("I live in " + city + ".");
// Output: I live in Portland.
```

---

## Combining Text: String Interpolation

**String interpolation** is a cleaner way to combine text and variables.

Syntax: `$"Text {variable} more text"`

```csharp
string name = "Nikki";
Console.WriteLine($"Hello, {name}!");
// Output: Hello, Nikki!
```

**Rules:**
- Put `$` before the opening quote
- Wrap variables in curly braces: `{variableName}`
- Much more readable than concatenation!

## Concatenation vs. Interpolation

Both do the same thing, but interpolation is usually easier to read.

### Concatenation (old way)

```csharp
string name = "Nikki";
string city = "Portland";
Console.WriteLine("Hi, I'm " + name + " from " + city + ".");
```

### Interpolation (modern way)

```csharp
string name = "Nikki";
string city = "Portland";
Console.WriteLine($"Hi, I'm {name} from {city}.");
```

**When to use each:**
- **Interpolation:** Most of the time (cleaner, easier to read)
- **Concatenation:** When you're learning or when building strings piece by piece

## Common Mistakes

### Forgetting the `$` in Interpolation

❌ **Wrong:**
```csharp
string name = "Nikki";
Console.WriteLine("Hello, {name}");  // Prints: Hello, {name}
```

✅ **Correct:**
```csharp
string name = "Nikki";
Console.WriteLine($"Hello, {name}");  // Prints: Hello, Nikki
```

---

### Using Quotes Around Variables

❌ **Wrong:**
```csharp
string name = "Nikki";
Console.WriteLine("name");  // Prints: name (literal text)
```

✅ **Correct:**
```csharp
string name = "Nikki";
Console.WriteLine(name);  // Prints: Nikki (variable value)
```

---

### Forgetting Spaces in Concatenation

❌ **Wrong:**
```csharp
Console.WriteLine("Hello" + name);  // Prints: HelloNikki
```

✅ **Correct:**
```csharp
Console.WriteLine("Hello " + name);  // Prints: Hello Nikki
```

Or use interpolation:
```csharp
Console.WriteLine($"Hello {name}");  // Prints: Hello Nikki
```

## Variable Naming Rules

**Rules (must follow):**
- Start with a letter or underscore: `name`, `_count`
- Can contain letters, numbers, underscores: `name2`, `first_name`
- Cannot contain spaces or special characters: `my-name` ❌
- Cannot be a C# keyword: `class`, `int`, `string` ❌

**Conventions (should follow):**
- Use **camelCase** for local variables: `firstName`, `totalCount`
- Use meaningful names: `name` not `x`, `age` not `a`
- Don't abbreviate excessively: `userName` not `usrNm`

### Examples

✅ **Good:**
```csharp
string firstName = "Nikki";
int age = 20;
double gradePointAverage = 3.8;
```

❌ **Bad:**
```csharp
string n = "Nikki";         // Too short
int a = 20;                 // Unclear
double gpa2023 = 3.8;       // Unnecessary detail
```

## Key Takeaways

✅ **Variables** store data in named containers

✅ Syntax: `type name = value;` (e.g., `string name = "Nikki";`)

✅ Common types: `string` (text), `int` (whole numbers), `double` (decimals), `bool` (true/false)

✅ Use `\n` for newlines, `\t` for tabs

✅ **Concatenation:** Join strings with `+`

✅ **Interpolation:** Use `$"text {variable}"` (cleaner!)

✅ Variable names: start with letter, use camelCase, be descriptive
