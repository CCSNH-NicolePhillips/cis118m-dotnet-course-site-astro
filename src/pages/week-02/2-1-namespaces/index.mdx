## üì¶ What is a Namespace?

A **namespace** is a container that organizes your code and prevents naming conflicts.

Think of it like a **postal system**:
- "John Smith" might be a common name
- But "John Smith, 123 Oak Street, Boston, MA" is unique
- The address (namespace) makes the name unambiguous

```csharp
namespace Airline.Reservations
{
    class Customer { /* booking info */ }
}

namespace Airline.Loyalty
{
    class Customer { /* points, status */ }
}

// Both "Customer" classes can coexist because they're in different namespaces
```

### Real-World Example: Amadeus GDS

The Amadeus Global Distribution System handles **millions** of flight bookings daily. Their codebase includes:

| Namespace | Purpose |
|-----------|---------|
| `Amadeus.Flights.Search` | Finding available flights |
| `Amadeus.Flights.Booking` | Reserving seats |
| `Amadeus.Flights.Pricing` | Calculating fares |
| `Amadeus.Hotels.Search` | Finding hotels |
| `Amadeus.Hotels.Booking` | Reserving rooms |

Without namespaces, a `Search` class in Flights would collide with `Search` in Hotels.

## üèóÔ∏è What is a Class?

A **class** is a blueprint that contains your code logic. Every piece of executable code in C# lives inside a class.

Think of it like an **architectural blueprint**:
- The blueprint (class) defines what a house could be
- The actual house (object) is built from that blueprint
- You can build many houses from one blueprint

```csharp
namespace MyProgram
{
    class Program  // The blueprint
    {
        // All your code goes inside here
    }
}
```

### The Minimum Structure

Every C# program requires at least:

```csharp
namespace ProgramName      // 1. Container for organization
{
    class Program          // 2. Blueprint for logic
    {
        static void Main() // 3. Entry point (covered in 2.2)
        {
            // Your code here
        }
    }
}
```

## üí° The `using` Directive

Instead of typing the full namespace path every time, you can import it:

```csharp
// Without using:
System.Console.WriteLine("Hello!");

// With using:
using System;
Console.WriteLine("Hello!");
```

The `using` directive at the top of your file says: *"I'll be using classes from this namespace, so let me reference them by short name."*

### Common Using Directives

```csharp
using System;                    // Console, Math, String, etc.
using System.Collections.Generic; // List<T>, Dictionary<K,V>
using System.IO;                 // File operations
using System.Linq;               // Query operations
```
