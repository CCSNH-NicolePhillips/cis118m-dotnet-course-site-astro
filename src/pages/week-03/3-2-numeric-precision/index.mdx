## üßÆ int: The Counting Type

Use `int` when you need **whole numbers** without decimals:

```csharp
int userCount = 1500;        // ‚úÖ Perfect for counting
int maxAttempts = 3;         // ‚úÖ Perfect for limits
int temperature = 72;        // ‚úÖ Good enough for room temp

int price = 19.99;           // ‚ùå COMPILER ERROR - can't store decimals
```

### int Facts:
- **Range:** -2,147,483,648 to 2,147,483,647
- **Memory:** 4 bytes (32 bits)
- **Speed:** Fastest numeric operations
- **Use for:** Counts, IDs, indexes, ages, quantities

## üìê double: The Scientific Type

Use `double` when you need **decimal precision** for calculations:

```csharp
double distance = 384400.5;   // ‚úÖ Great for measurements
double pi = 3.14159265359;    // ‚úÖ Mathematical constants
double velocity = 299792.458; // ‚úÖ Scientific data

double price = 19.99;         // ‚ö†Ô∏è Works, but NOT for money!
```

### double Facts:
- **Range:** ¬±5.0 √ó 10^‚àí324 to ¬±1.7 √ó 10^308
- **Memory:** 8 bytes (64 bits)
- **Precision:** ~15-16 significant digits
- **Use for:** Science, graphics, physics, measurements

## üí∞ decimal: The Money Type

<div class="critical-warning">
  <h3>‚ö†Ô∏è CRITICAL: Never Use double for Money!</h3>
  <p>
    Binary floating-point (double) cannot accurately represent many decimal fractions. 
    This leads to <strong>rounding errors</strong> that accumulate over time.
  </p>
</div>

### The Famous 0.1 + 0.2 Problem

<TryItNowRunner 
  starterId="week-03-float-problem" 
  height={300}
  code={`using System;

class Program
{
    static void Main()
    {
        // The dangerous way (double)
        double badPrice = 0.1 + 0.2;
        Console.WriteLine("double: 0.1 + 0.2 = " + badPrice);
        
        // The safe way (decimal)
        decimal goodPrice = 0.1m + 0.2m;
        Console.WriteLine("decimal: 0.1 + 0.2 = " + goodPrice);
    }
}`}
  instructions="Run this and see the difference! The 'm' suffix makes it a decimal literal." 
/>

### Why This Happens

<DeepDiveTabs
  title="Binary vs Decimal Arithmetic"
  tabs={[
    {
      id: "binary",
      label: "üî¢ The Problem",
      content: `
        <h3>Binary Floating-Point</h3>
        <p>Computers store numbers in binary (base 2). Just like 1/3 = 0.333... in decimal, some decimal fractions become infinite in binary.</p>
        <p><strong>0.1 in binary:</strong> 0.0001100110011001100110011... (repeats forever)</p>
        <p>The computer has to round, and those tiny errors add up in financial calculations!</p>
      `
    },
    {
      id: "solution",
      label: "‚úÖ The Solution",
      content: `
        <h3>Decimal Type</h3>
        <p>The <code>decimal</code> type uses base-10 internally, so it represents decimal fractions exactly.</p>
        <ul>
          <li><strong>128-bit precision</strong> ‚Äî 28-29 significant digits</li>
          <li><strong>Exact decimal math</strong> ‚Äî No binary rounding errors</li>
          <li><strong>Required for finance</strong> ‚Äî Banks, e-commerce, payroll</li>
        </ul>
        <p><strong>Note:</strong> Use the <code>m</code> suffix for decimal literals: <code>45.99m</code></p>
      `
    }
  ]}
/>

### Using decimal Correctly

```csharp
decimal productPrice = 45.99m;     // Note the 'm' suffix!
decimal taxRate = 0.0725m;
decimal totalCost = productPrice * (1 + taxRate);

Console.WriteLine("Total: $" + totalCost);  // Exact to the penny
```
