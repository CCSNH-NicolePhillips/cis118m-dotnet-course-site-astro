## ğŸ—ï¸ What is a Variable?

A **variable** is a named storage location in your computer's memory (RAM). Think of it as a labeled box that holds a single piece of data.

<div class="concept-highlight">
  <h3>The Parking Lot Analogy</h3>
  <p>Imagine your computer's memory as a giant parking garage:</p>
  <ul>
    <li><strong>Variable Name</strong> = The parking spot number (e.g., "A-42")</li>
    <li><strong>Variable Type</strong> = What kind of vehicle can park there (car, motorcycle, truck)</li>
    <li><strong>Variable Value</strong> = The actual vehicle parked in that spot</li>
  </ul>
</div>

## ğŸ” Anatomy of a Variable Declaration

```csharp
int  userCount  =  1500;
â”‚       â”‚       â”‚    â”‚
â”‚       â”‚       â”‚    â””â”€â”€ Value (the data stored)
â”‚       â”‚       â””â”€â”€ Assignment operator
â”‚       â””â”€â”€ Identifier (the name)
â””â”€â”€ Type (what kind of data)
```

### The Four Parts:

| Component | Description | Example |
|-----------|-------------|---------|
| **Type** | What kind of data can be stored | `int`, `string`, `bool` |
| **Identifier** | The name you give the variable | `userCount`, `firstName` |
| **Assignment** | The `=` operator stores the value | `=` |
| **Value** | The actual data being stored | `1500`, `"Alice"`, `true` |

  <div class="rule invalid">
    <h4>âŒ Invalid Names</h4>
    <ul>
      <li><code>1stPlace</code> â€” Can't start with number</li>
      <li><code>user-count</code> â€” No hyphens</li>
      <li><code>first name</code> â€” No spaces</li>
      <li><code>class</code> â€” Reserved keyword</li>
      <li><code>int</code> â€” Reserved keyword</li>

### Naming Conventions (Best Practices)

| Style | Use For | Example |
|-------|---------|---------|
| **camelCase** | Local variables, parameters | `userCount`, `firstName` |
| **PascalCase** | Methods, classes, properties | `GetUserCount`, `FirstName` |
| **UPPER_CASE** | Constants | `MAX_CONNECTIONS`, `PI` |

## ğŸ”„ Reassignment

Once a variable is declared, you can change its value (unless it's a constant):

  starterId="week-03-reassignment"
