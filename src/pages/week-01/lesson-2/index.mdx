## What Are Variables?

A **variable** is a container that holds a value. Think of it like a labeled box:

- The **name** is the label on the box
- The **value** is what's inside the box
- The **type** tells you what KIND of thing can go in the box

```csharp
string name = "Alice";  // name holds "Alice"
int age = 20;           // age holds 20
```

## String Variables

A **string** is text (letters, numbers, symbols). Strings must be in quotes.

```csharp
string city = "Boston";
string favoriteFood = "pizza";
string greeting = "Hello!";
```

## Integer Variables (Numbers)

An **int** (integer) holds whole numbers (no decimals).

```csharp
int age = 20;
int year = 2024;
int score = 100;
```

You can do math with numbers:

```csharp
int age = 20;
int nextYearAge = age + 1;  // 21
```

## Printing with Labels (Formatted Output)

Make your output readable by adding labels:

```csharp
Console.WriteLine("Name: Alice");
Console.WriteLine("City: Boston");
```

Output:
```
Name: Alice
City: Boston
```

This is much better than just printing "Alice" and "Boston" without context!

### Task 3: String concatenation vs interpolation

There are **two ways** to combine text and variables:

**1) Concatenation** (using `+`):
```csharp
string name = "Alice";
Console.WriteLine("Hello, " + name + "!");
```

**2) Interpolation** (using `$` and `{}`):
```csharp
string name = "Alice";
Console.WriteLine($"Hello, {name}!");
```

Both produce: `Hello, Alice!`

**Interpolation is usually easier to read** — that's what we recommend!

## Common Mistakes

❌ **Forgetting quotes around strings:**
```csharp
string name = Alice;  // ERROR - need quotes
```

✅ **Correct:**
```csharp
string name = "Alice";  // WORKS
```

---

❌ **Using int for text:**
```csharp
int name = "Alice";  // ERROR - strings aren't ints
```

✅ **Correct:**
```csharp
string name = "Alice";  // WORKS
```

---

❌ **Forgetting $ before interpolated strings:**
```csharp
string name = "Alice";
Console.WriteLine("{name}");  // Prints {name} literally
```

✅ **Correct:**
```csharp
string name = "Alice";
Console.WriteLine($"{name}");  // Prints Alice
```

---

❌ **Using + instead of , in variable declaration:**
```csharp
string first = "Bob", last = "Smith";  // WORKS but confusing
```

✅ **Better:**
```csharp
string first = "Bob";
string last = "Smith";  // Clearer
```

## Variable Naming Rules

✅ **Good names:**
- `firstName` (camelCase - lowercase first word, uppercase others)
- `age`
- `totalScore`

❌ **Bad names (won't compile):**
- `first name` (no spaces)
- `2ndPlace` (can't start with number)
- `class` (reserved keyword)

❌ **Bad names (compile but confusing):**
- `x`, `y`, `z` (not descriptive)
- `asdf` (meaningless)
- `thing1`, `thing2` (vague)

**Pro tip:** Use names that explain what the variable holds. Future you will thank you!

## Blank Lines

You can print a blank line with no text:

```csharp
Console.WriteLine();  // Prints empty line
```

This is useful for spacing out your output:

```csharp
Console.WriteLine("=== Profile ===");
Console.WriteLine();
Console.WriteLine($"Name: {name}");
Console.WriteLine($"Age: {age}");
```

## Key Takeaways

✅ **Variables** store values (like labeled boxes)

✅ **string** holds text (in quotes): `"Hello"`

✅ **int** holds whole numbers (no quotes): `42`

✅ Use `+` to concatenate: `"Hello, " + name`

✅ Use `$""` and `{}` to interpolate: `$"Hello, {name}"`

✅ **Interpolation is easier to read** (recommended!)

✅ Label your output so it makes sense: `Console.WriteLine($"Name: {name}");`

✅ Use descriptive variable names (not `x`, `y`, `z`)
