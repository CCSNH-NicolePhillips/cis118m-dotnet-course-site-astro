## Console Applications

A **console app** is a text-based program that runs in a terminal window. It:
- Prints text output to the screen
- Can accept text input from the user
- Has no graphical interface (no buttons, windows, etc.)

**Why start with console apps?**

They're the simplest way to learn programming fundamentals. Later you'll learn to build apps with graphics, but console apps let you focus on logic first.

## Adding Comments

**Comments** are notes for humans. The computer ignores them completely.

### Try It Now #2: Two Lines + Comment

<TryItNowRunner
  code={"using System;\n\nclass Program\n{\n    static void Main()\n    {\n        // This is a comment (humans read it, computer ignores it)\n        Console.WriteLine(\"Line 1\");\n        Console.WriteLine(\"Line 2\");\n    }\n}"}
  starterId="week-01-lesson-1-comments"
  height={200}
  instructions="Run this code. Notice the comment line doesn't appear in the output."
/>

**Comment syntax:**

- `// Single line comment` ‚Äî Everything after `//` on that line is ignored
- `/* Multi-line comment */` ‚Äî Everything between `/*` and `*/` is ignored

**When to use comments:**
- Explain WHY you're doing something (not what‚Äîthe code shows what)
- Mark sections of code
- Temporarily disable code for testing

## Common Mistakes (and how to fix them)

### Missing Quotes

‚ùå **Wrong:**
```csharp
Console.WriteLine(Hello);  // ERROR
```

‚úÖ **Correct:**
```csharp
Console.WriteLine("Hello");  // WORKS
```

Text must always be in quotes!

---

### Missing Semicolon

‚ùå **Wrong:**
```csharp
Console.WriteLine("Hello")  // ERROR
```

‚úÖ **Correct:**
```csharp
Console.WriteLine("Hello");  // WORKS
```

Every statement ends with a semicolon.

---

### Typos in Keywords

‚ùå **Wrong:**
```csharp
Console.WriteLin("Hello");  // ERROR (missing 'e')
```

‚úÖ **Correct:**
```csharp
Console.WriteLine("Hello");  // WORKS
```

Spelling matters! C# is case-sensitive.
