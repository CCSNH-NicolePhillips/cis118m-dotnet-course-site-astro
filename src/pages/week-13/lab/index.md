# Week 13 Lab: Persistent To-Do List (file-based)

## Overview
Upgrade your to-do list so it saves to a file and loads when the program starts.

## Requirements checklist (acceptance criteria)
- [ ] Uses a `List<string>` for tasks
- [ ] Loads tasks from a file if it exists
- [ ] Lets user add at least 1 task
- [ ] Saves tasks back to file (WriteAllLines is fine)
- [ ] Prints the updated list
- [ ] Runs without crashing even if file is missing
